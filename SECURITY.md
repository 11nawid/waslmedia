# Security Policy

## Supported security posture

This repository aims to keep:

- secrets out of version control
- dependencies reasonably up to date
- CI checks running on pull requests
- automated dependency and code scanning enabled through GitHub workflows

## Reporting a vulnerability

Please do not post unpatched security issues publicly in GitHub issues.

Instead:

1. Contact the maintainer directly with a clear description, impact, and reproduction steps.
2. Include affected files, endpoints, or setup details if known.
3. Share only the minimum evidence needed and never include live credentials.

Current contact path:

- Instagram DM: [@11.skibidi](https://www.instagram.com/11.skibidi/)

## What to include

- vulnerability summary
- affected area
- attack scenario
- reproduction steps
- suggested fix if you have one

## Response goals

- acknowledge reports as quickly as possible
- confirm severity and scope
- patch or mitigate when reproducible
- credit responsible reports when appropriate
